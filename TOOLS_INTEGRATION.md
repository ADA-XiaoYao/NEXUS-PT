## Tool Execution Framework

### Tool Adapter Architecture

NEXUS-PT Pro uses a unified adapter pattern to execute tools:

```typescript
// Example: Execute XBOW
const context: ExecutionContext = {
  projectId: 1,
  taskId: "task-123",
  userId: 1,
  parameters: {
    targetUrl: "https://example.com",
    scope: "full"
  }
};

const result = await toolAdapterFactory.executeTool("xbow", context);
// Returns: ToolResponse with vulnerabilities, findings, etc.
```

## Tool Orchestration

NEXUS-PT Pro orchestrates multiple tools in sequence:

1. **Reconnaissance Phase**: OSINT tools, Shodan, VirusTotal
2. **Scanning Phase**: Nmap, Nuclei, API scanners
3. **Vulnerability Analysis**: XBOW, Hadrian, Aikido
4. **Exploitation Phase**: Metasploit, SQLMap, custom exploits
5. **Post-Exploitation**: Villager, PentAGI, custom modules
6. **Reporting**: Compliance reports, breach simulation results

## Tool Configuration

### API Keys and Credentials

Tools requiring API keys:
- XBOW
- Escape
- NodeZero
- RidgeBot
- Mindgard
- Shodan
- VirusTotal

Configuration via environment variables:
```bash
XBOW_API_KEY=xxx
ESCAPE_API_KEY=xxx
NODEZERO_API_KEY=xxx
# ... etc
```

### Tool-Specific Parameters

Each tool supports custom parameters:
```typescript
{
  toolId: "xbow",
  parameters: {
    targetUrl: "https://example.com",
    depth: "full",
    timeout: 3600,
    reportFormat: "json"
  }
}
```
